# Shared Development Guidelines

All projects under ~/git/ follow these principles.

## Core Principles
- Write minimal, focused changes
- Prefer simple solutions over clever ones
- Never silently swallow errors

## Available Skills
- `/coding-standards` - General code quality
- `/python-patterns` - Python idioms and best practices
- `/python-testing` - pytest and TDD for Python
- `/frontend-patterns` - React/Next.js patterns
- `/backend-patterns` - API and database patterns
- `/tdd-workflow` - Test-driven development process
- `/commit-message` - Conventional commit messages
- `/svelte-frontend-patterns` - Svelte 5 / SvelteKit 2 patterns
- `/skill-creator` - Create and improve skills
